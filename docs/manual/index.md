## 二、 软件能做什么工作

NebulaLab 旨在彻底缩短从“原始实验仪器数据”到“学术报告/论文出版插图”之间的繁琐路径。它主要承担以下核心工作：

- **数据导入与整理**：一键拖拽导入 CSV、TXT、Excel、DAT、XY、ASC 等十余种常规仪器数据。支持多 Sheet 解析与“分组树”样品组织。
- **一站式数据预处理**：提供直观的单元格双击编辑，内置数据裁剪、平滑降噪（如 SG 滤波器）、基线扣除与归一化等高频算法，秒级过滤噪声。
- **科学绘图与无损导出**：支持多曲线同屏对比，高自由度调整曲线粗细、图例、网格和坐标轴。一键导出论文级的无损 SVG 矢量图或高清晰 PNG。
- **工作流模板复用**：成熟的数据清洗与绘图配置可一键保存为模板。面对新数据时，套用模板即可在数秒内全自动重现并批量执行处理过程。
- **社区与插件生态**：共享与下载高价值科研资源。一键在软件内检索并安装官方或社区共享的实用插件、精美主题与高复用工作流模板，促进团队高效协作。
- **课题组私有云部署**：支持课题组局域网自建数据服务器部署。方便课题组成员间数据的一键云端备份、多终端同步与协同编辑，保障实验数据安全隐私。

## 三、 在哪里下载与安全放行

> [!IMPORTANT]
> **官方唯一下载渠道**
> 为了保障软件的纯粹性与安全性，我们目前未向商业机构购买昂贵的高级代码签名证书。**请放心，软件本身完全开源且安全无毒。** 请务必通过以下官方 Releases 页面获取最新版安装包：
>
> 🔗 **[点击前往官方 GitHub Releases 下载最新版本](https://github.com/TshyGO/NebulaLab-Releases/releases/latest)**

### 1. 浏览器下载拦截解除
由于缺乏商业签名，部分浏览器在下载时可能会弹出拦截提示：
- **Edge 浏览器**：若提示“无法安全下载”，请在下载栏鼠标悬停于该项，点击右侧的 **“...” (显示更多操作)** -> 选择 **“保留”** -> 点击 **“显示详细信息”** -> 点击 **“仍然保留”**。
- **Chrome 浏览器**：若下载被阻止，请点击 **“保留”** -> 并在弹出的二次确认中点击 **“保留危险文件”** 完成下载。

### 2. 首次运行系统拦截解除
首次在 Windows 中运行安装包或 `NebulaLab.exe` 时，系统可能会弹出蓝色警告窗口提示“Windows 已保护您的电脑”：
1. 点击窗口文本正中部的 **“更多信息”** 链接。
2. 窗口右下方会显现隐藏的 **“仍要运行”** 按钮。
3. 点击 **“仍要运行”** 即可正常启动，此后再次打开该版本将不会再弹出拦截提示。

## 四、 故障排查与反馈渠道

如果您在使用过程中遇到了崩溃、文件解析失败或算法异常，可以通过以下渠道获得我们的协助：

### 1. 如何定位软件日志（Logs）
在向我们提交问题反馈前，附带上软件生成的日志文件可以帮助开发团队以最快速度定位问题：
- 日志存放路径位于软件安装目录（或免安装版解压目录）下的 `logs/` 文件夹内。
- 文件名通常为当前日期，例如 `2026-06-01.log`。
- > [!TIP]
  > **隐私与脱敏安全**
  > 在发送日志给他人前，请务必打开日志检查，可将其中敏感的本地文件名、文件夹绝对路径或个人隐私数据进行手动脱敏处理（例如替换为占位符）。

### 2. 官方异步反馈渠道（GitHub Issues）
如果您发现了非保密的公开 Bug，或者有很好的功能建议，我们极力推荐您通过 GitHub Issues 进行反馈：
- **提交地址**：[NebulaLab-Releases Issues](https://github.com/TshyGO/NebulaLab-Releases/issues)
- **提交规范**：
  1. 请选择适合的 Issue 模板（如“Bug 汇报”或“功能建议”）。
  2. 描述您的**预期行为**与**实际行为**。
  3. 提供可以复现该故障的最简操作步骤。
  4. 如果可以，上传脱敏后的最简样例数据或报错截图。

### 3. 官方私密支持邮箱
如果您购买了 Pro 专业版，或者您的故障涉及尚未发表的敏感科研实验数据、或是课题组内部的保密商务合作：
- **私密支持信箱**：`support@nebulalab.io` *(仿真服务邮箱，保护隐私)*
- 专属客服和算法专家将通过邮件或安全专线与您取得联系，保障您的科研隐私与信息安全。
